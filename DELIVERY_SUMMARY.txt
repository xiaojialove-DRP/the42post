📡 API ENDPOINTS IMPLEMENTED

Authentication:
  POST   /api/auth/register              — Register new user
  POST   /api/auth/login                 — Login with email/password
  GET    /api/auth/verify/:token         — Verify email
  GET    /api/auth/me                    — Get current user (protected)
  PATCH  /api/auth/me                    — Update profile (protected)

Forge Workflow:
  POST   /api/forge/probe                — Generate intuition probe (Claude API)
  POST   /api/forge/generate             — Generate five-layer skill (Claude API)

Skills Management:
  GET    /api/skills                     — List published skills
  GET    /api/skills/:skill_id           — Get skill detail with manifest
  POST   /api/skills                     — Create & publish skill (protected)
  PATCH  /api/skills/:skill_id           — Update skill (protected, author only)
  DELETE /api/skills/:skill_id           — Delete skill (protected, author only)
  GET    /api/skills/:skill_id/manifest  — Get manifest (for verification)
  POST   /api/skills/:skill_id/sign      — Add covenant signature (protected)
  GET    /api/skills/user/skills         — Get author's skills (protected)

Search & Discovery:
  GET    /api/search                     — Full-text search
  GET    /api/search/trending            — Trending skills
  GET    /api/search/domain/:domain      — Skills by domain

Agents (Stub):
  POST   /api/agents/bind                — Bind agent (protected)
  GET    /api/agents/:agent_id/capabilities — Get agent capabilities
  POST   /api/agents/:agent_id/verify    — Verify binding (protected)

Health:
  GET    /health                         — Server health check

🗄️ DATABASE SCHEMA

Key Features:

🔄 WORKFLOW: From Idea to Published Skill

STEP 0: Choose Forge Mode
  ├─ Shadow Agent: "I have an idea"
  └─ Direct Knight: "I have an Agent"

STEP 1: Register & Account Linking
  └─ Email + username + account_type → copyright ownership

STEP 2a: Generate Intuition Probe
  └─ POST /api/forge/probe
     └─ Claude API generates: scenario, thesis, antithesis, extreme

STEP 2b: Select & Generate Skill
  └─ POST /api/forge/generate
     └─ Claude API generates complete five-layer structure

STEP 3: Refine & Review
  └─ Review skill, optionally update boundaries & use cases

STEP 4: Publish with Manifest
  └─ POST /api/skills
     └─ Generate soul-hash
     └─ Create manifest with author signature
     └─ Store in database
     └─ Return skill_id, soul_hash, manifest

OPTIONAL: Multi-stakeholder Approval
  └─ POST /api/skills/:skill_id/sign
     └─ Add covenant signatures (collective approval)
