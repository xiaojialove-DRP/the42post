## 🎯 Quick FAQ

**Q：我没有技术背景，能创建Skill吗？**  
A：完全可以！遵循Path A，只需要README + GETTING_STARTED + SKILL_DESIGN_GUIDE + EXAMPLES

**Q：我该读RESEARCH_FOUNDATION.md吗？**  
A：可选。如果你想理解"为什么"这样设计，建议读。否则可以跳过。

**Q：多少时间能创建一个Skill？**  
A：学习 2-3 小时，创建 30分钟-2小时（取决于复杂度）

**Q：为什么需要五层？不能简化吗？**  
A：五层确保你的Skill"可信"。每层都有学术基础（见RESEARCH_FOUNDATION.md）

**Q：能只学某几层吗？**  
A：不建议。五层是一个完整系统 - DEFINING没有VALIDATING就无法验证；没有CONTEXTUALIZING就无法跨文化适用。

## 💡 Key Concepts to Master

当你完成学习时，你应该理解以下概念：

### 五层架构 (Five-Layer Architecture)
```
DEFINING      → 核心价值观或原则是什么？
INSTANTIATING → 这在实际中如何体现？
FENCING       → 何时应用？何时不应用？
VALIDATING    → 如何验证是否遵循？
CONTEXTUALIZING → 如何跨文化适配？
```

### Soul-Hash（技能的不可伪造身份）
```
SOUL_[hash]_[timestamp]
由以下生成：skill content + author email + timestamp
防篡改，任何人都能验证真实性
```

### Manifest Covenant（多方签名）
```json
{
  "soul_hash": "SOUL_...",
  "author": { "email": "..." },
  "signatures": {
    "author": "signature",
    "reviewer": "signature"
  }
}
```
不是单人决定，而是多个利益相关者的共识

### 语义资本 (Semantic Capital)
一个Skill的质量取决于：
- 清晰性 - 不同背景的人能理解吗？
- 跨越性 - 能在不同文化中被理解吗？
- 可验证性 - 有明确的test case吗？
- 复用性 - 能在多个场景中应用吗？
- 社区认可 - 被多个stakeholder批准吗？
