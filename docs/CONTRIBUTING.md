# Contributing Guidelines

Thank you for contributing to THE 42 POST!

## Code of Conduct

- Be respectful and inclusive
- Welcome diverse perspectives
- Focus on ideas, not people

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make changes following the style guide
4. Commit with clear messages
5. Push and create a pull request

## Commit Message Format

```
type(scope): subject

body

footer
```

### Types
- `feat` - New feature
- `fix` - Bug fix
- `docs` - Documentation
- `style` - Formatting
- `refactor` - Code reorganization
- `test` - Tests
- `chore` - Build/dependencies

### Example
```
feat(forge): add intuition probe generation

Adds new endpoint to generate intuition probes for skill creation.

Closes #123
```

## Code Style

- **JavaScript**: Modern ES6+, arrow functions, const by default
- **HTML**: Semantic markup
- **CSS**: BEM naming convention
- **Comments**: Explain "why", not "what"

## Testing

- Test locally before submitting PR
- No console errors or warnings
- Test in multiple browsers
- Test on mobile devices

## Pull Request Process

1. Update documentation
2. Link to related issues
3. Write clear PR description
4. Respond to feedback
5. Keep branch updated with main

## What to Work On

- Documentation improvements
- Bug fixes
- UI/UX enhancements
- Test coverage
- Performance optimization
