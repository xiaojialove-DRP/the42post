# 参与 THE 42 POST

THE 42 POST 是一个开放研究项目。我们欢迎各种形式的参与和协作。

## 🤝 我们寻找

### 💡 技能创建者（最简单）
在平台上直接设计并发布新技能。无需编码。
- 访问 [THE 42 POST](https://the42post-production.up.railway.app)
- 点击"技能锻造"创建您的第一个技能
- 发布并分享给社群

### 🎨 设计师和 UX 研究员
改进平台的界面、用户体验和可访问性。
- 报告设计问题：[GitHub Issues](https://github.com/xiaojialove-DRP/the42post/issues)
- 建议 UI/UX 改进
- 参与用户研究

### 🔧 开发者（后端 / 前端）
增强 API、改进技能生成、优化性能。
- Fork 仓库：`git clone https://github.com/yourusername/the42post.git`
- 创建特性分支：`git checkout -b feature/your-feature`
- 提交 Pull Request，描述清楚你的改进
- 遵循简洁的 commit 格式：`type(scope): subject`

**Commit 类型：**
- `feat` - 新功能
- `fix` - Bug 修复
- `docs` - 文档
- `refactor` - 代码重构
- `test` - 测试
- `style` - 格式化

**代码风格：**
- JavaScript: 使用 ES6+，优先用 const
- HTML: 语义化标签
- CSS: BEM 命名约定
- 注释：解释"为什么"，不是"是什么"

### 🔬 研究员
使用 THE 42 POST 进行学术研究，分析技能数据。
- 通过 API 访问技能数据：`/api/skills`, `/api/search`
- 获取 JSON 格式的五层结构
- 发布研究发现并引用 THE 42 POST
- 分享结果和见解回馈社群

### 📝 文档编写者
翻译文档、改进说明、补充示例。
- 翻译 README 到其他语言
- 改进现有文档的清晰度
- 添加使用示例和最佳实践

## 💬 代码审查与合作

- **尊重多样观点** — 不同背景的人看问题角度不同
