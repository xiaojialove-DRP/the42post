# Contributing to THE 42 POST

THE 42 POST is an open research project. We welcome contributions and collaboration in many forms.

## 🤝 Ways to Contribute

### 💡 Skill Creators (Easiest)
Design and publish new skills directly on the platform. No coding required.
- Visit [THE 42 POST](https://the42post-production.up.railway.app)
- Click "Skill Forge" to create your first skill
- Publish and share with the community

### 🎨 Designers & UX Researchers
Improve the platform's interface, user experience, and accessibility.
- Report design issues: [GitHub Issues](https://github.com/xiaojialove-DRP/the42post/issues)
- Suggest UI/UX improvements
- Participate in user research

### 🔧 Developers (Backend / Frontend)
Enhance the API, improve skill generation, optimize performance.
- Fork the repository: `git clone https://github.com/yourusername/the42post.git`
- Create a feature branch: `git checkout -b feature/your-feature`
- Submit a Pull Request with a clear description
- Follow clean commit format: `type(scope): subject`

**Commit Types:**
- `feat` - New feature
- `fix` - Bug fix
- `docs` - Documentation
- `refactor` - Code refactoring
- `test` - Tests
- `style` - Formatting

**Code Style:**
- JavaScript: Use ES6+, prefer const
- HTML: Semantic markup
- CSS: BEM naming convention
- Comments: Explain "why", not "what"

### 🔬 Researchers
Conduct academic research using THE 42 POST and analyze skill data.
- Access skill data via API: `/api/skills`, `/api/search`
- Get five-layer structure in JSON format
- Publish research findings and cite THE 42 POST
- Share insights and results with the community

### 📝 Documentation Contributors
Translate documents, improve clarity, add examples.
- Translate README to other languages
- Improve documentation clarity
- Add usage examples and best practices

## 💬 Code Review & Collaboration

Our principles:
- **Respect diverse perspectives** — Different backgrounds bring different insights
- **Focus on ideas, not people** — Feedback is technical, not personal
- **Welcome newcomers** — We all started somewhere
