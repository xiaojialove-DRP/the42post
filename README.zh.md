# THE 42 POST 🛸

**一个人人可参与锻造AI价值观的开放研究平台**

让普通人参与定义、分享和验证AI系统的价值观。

---

## 🎯 THE 42 POST 是什么？

THE 42 POST 是一个开放平台，让任何人都能创建"技能"（Skills）—— 人类价值观、原则和伦理准则的结构化、可验证的表示，用来改进AI的行为。

与其让AI的价值观隐含在训练数据中，THE 42 POST 将它们变得：
- **显式** — 价值观用五层框架清楚地定义
- **可共享** — 任何人都可以发现和使用社区创建的技能
- **可验证** — 每个技能都包含测试用例和验证标准
- **跨文化** — 技能包含针对不同文化的特定适配

### 核心功能

- 🛠️ **技能锻造工作坊** — 四步指导流程将您的想法转化为结构化技能
- 📚 **技能库** — 发现 42+ 由设计师、伦理学家和 AI 研究员创建的示例技能
- 👁️ **预览和审查** — 发布前查看完整的五层结构
- ✏️ **编辑和重新生成** — 基于 AI 反馈精细调整技能
- 🤖 **AI 集成** — 技能可以集成到 AI 智能体和系统中
- 📊 **使用分析** — 追踪您的技能何时以及如何被采用
- 🌍 **开源** — MIT 许可证，基于 Node.js 和开放数据

## ❓ 为什么我们构建这个？

### 问题所在

当前 AI 系统存在关键缺陷：
- **隐含价值观** — AI 的价值观隐藏在训练数据中，无法审计
- **文化错位** — 同一个 AI 在不同文化中的行为不一致
- **无法验证** — 用户无法确认 AI 真正遵循什么原则
- **集中控制** — 价值观决策由少数组织做出

### 我们的解决方案

我们创建 THE 42 POST 来：

1. **民主化 AI 价值观对齐** — 给设计师、伦理学家和领域专家在 AI 价值观中发言
2. **让价值观透明** — 用显式、可审计的原则替代黑箱对齐
3. **启用社区验证** — 使用带有测试用例的结构化技能，任何人都可以验证
4. **支持文化多样性** — 让不同社区将原则适配到其背景中

## 🚀 如何使用 THE 42 POST

### 快速开始（所有人）

1. **访问平台**：打开 http://localhost:8080（或您部署的实例）
2. **浏览技能库**：探索 42+ 由设计师、伦理学家和研究员创建的示例技能
3. **从示例学习**：阅读现有技能的五层结构以理解方法
4. **尝试影子智能体**：体验技能如何在对话中影响 AI 行为
5. **开始创建**：点击"技能锻造"创建您的第一个技能

### 技能发现者（使用技能）

1. **搜索库** — 按领域、创建者或关键词查找技能
2. **阅读技能详情** — 每个技能包括：
   - 创建者的动机和故事
   - 五层结构（定义、场景、边界、验证、文化适配）
   - 真实世界的使用示例
   - 社区反馈和评分
3. **体验技能** — 使用"影子智能体"看技能如何引导 AI 行为
4. **提供反馈** — 评价技能并建议改进
5. **分享技能** — 在自己的 AI 项目中使用或推荐给他人

### 创建者（锻造技能）

1. **启动锻造工作坊**：点击"技能锻造"进入四步流程：

   **步骤 1️⃣  想法**（2-3 分钟）
   - 输入用户名和邮箱
   - 描述您的核心想法或价值原则
   - 选择最适合您想法的创意维度

   **步骤 2️⃣  生成**（1-2 分钟）
   - 观看系统生成您的五层技能结构：
     - **定义**：核心原则声明
     - **场景**：原则的真实世界示例
     - **边界**：何时应用和何时不应用
     - **验证**：验证原则的测试用例
     - **文化适配**：如何在不同文化中适配
   - 清晰的视觉反馈显示锻造过程

   **步骤 3️⃣  审查**（3-5 分钟）
   - 使用所有五层预览您的完整技能
   - 根据需要编辑技能名称和定义
   - 可选地提供反馈以重新生成任何部分
   - 在交互式预览模式框中查看完整结构

   **步骤 4️⃣  发布**（1 分钟）
   - 选择许可选项（商业用途、混搭权利）
   - 接受创建者契约
   - 发布到库
   - 获得您永久的 Soul-Hash 身份
   - 下载您的纪念创建者卡
   - 接收确认邮件

2. **发布后**：
   - 您的技能被添加到公共库
   - 监控谁在使用您的技能
   - 接收来自研究员和 AI 开发者的反馈
   - 根据真实世界的使用迭代您的技能

### 研究员（研究和验证技能）

研究员可以使用 THE 42 POST 研究 AI 系统中的人类价值观，并验证结构化技能定义的有效性。

**研究工作流**：

1. **探索技能库**
   - 浏览跨不同领域的 42+ 示例技能
   - 分析每个技能的五层结构
   - 理解不同创建者如何处理价值观定义
   - 将技能数据导出为 JSON 格式进行分析

2. **研究技能有效性**
   - 使用"影子智能体"测试技能如何影响 AI 行为
   - 针对验证层运行自定义测试用例
   - 比较同一原则在不同文化中的具体化方式
   - 测量意图价值观和实际 AI 输出之间的对齐

3. **验证技能质量**
   - 使用边界层测试边界情况
   - 验证文化适配层是否捕捉文化细微差别
   - 对技能清晰性和适用性提供反馈
   - 通过反馈系统建议改进

4. **进行比较研究**
   - 分析一个领域中多个技能的模式
   - 比较不同技能创建者如何处理类似问题
   - 研究技能迭代的演变
   - 提取关于跨文化价值优先级的见解

5. **发布研究发现**
   - 根据发现锻造研究导向的技能
   - 在技能描述中记录案例研究
   - 为您的研究领域创建参考技能
   - 贡献于不断增长的人类中心 AI 研究体系

**研究 API 访问**：
- 通过 REST API 访问技能数据（`/api/skills`, `/api/search`）
- 获取 JSON 格式的完整五层结构
- 按领域、创建者或标签查询技能
- 下载批量技能进行大规模分析

**示例研究问题您可以回答**：
- 创建者如何一致地跨文化定义价值观？
- 什么使技能的测试用例更有效？
- 不同领域的技能结构有何不同？
- 哪些文化适配对全球 AI 最重要？

## 🛠️ 本地开发入门

### 先决条件
- Node.js 18+
- SQLite 或 PostgreSQL
- npm

### 安装

```bash
# 克隆仓库
git clone https://github.com/xiaojialove-DRP/the42post.git
cd the42post

# 安装后端依赖
cd backend
npm install

# 配置环境
cp .env.example .env
# 使用以下设置编辑 .env：
# - DATABASE_URL (本地 SQLite 或远程 PostgreSQL)
# - ANTHROPIC_API_KEY (用于技能生成)
# - FRONTEND_URL (通常是 http://localhost:8080)

# 运行后端
npm run dev    # 使用 nodemon 的开发
# 或
npm start      # 生产环境

# 在另一个终端中，打开前端
# 前端提供在 http://localhost:8080/
```

### 环境设置

在 `backend/` 目录中创建 `.env` 文件：

```env
# 数据库
DATABASE_URL=sqlite:./database.sqlite3
# 对于 PostgreSQL: postgresql://user:password@localhost:5432/42post

# 认证
JWT_SECRET=your-super-secret-key-minimum-32-characters
JWT_EXPIRY=24h

# Claude AI（用于技能生成）
ANTHROPIC_API_KEY=sk-ant-your-actual-key-here

# 邮件服务（可选 - 开发模式下移除）
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password (不是常规密码！)
SMTP_SECURE=false

# 服务器
PORT=3000
NODE_ENV=development
FRONTEND_URL=http://localhost:8080
```

### 验证设置

```bash
# 测试后端健康状态
curl http://localhost:3000/health

# 测试前端
访问 http://localhost:8080 在浏览器中
```
